## 2. 产品目标

以 Quill 的本地优先架构为底座，制作一款面向英语原著阅读的个人桌面阅读器：阅读过程安静、书籍感强，AI 能在不打断阅读的前提下完成语境查词、释义追问、书中理解讨论，并把查询结果沉淀为可回看、可标记、可复习的学习资料。

核心闭环：

```text
阅读原文 -> 语境查词/解释 -> 继续追问 -> 收藏或保留历史
         -> 正文回显 -> 学习状态更新 -> 回到原文复习
```

成功标准不是“堆叠更多 AI 功能”，而是减少在阅读器、词典、ChatGPT 和生词本之间切换的次数。

## 3. 已确认的产品方向

### 3.1 阅读视觉采用 Anx Reader 的气质

- 主阅读主题使用温暖但不过黄的纸张底色、深墨色正文和低对比度边界。
- 英文正文优先使用适合长时间阅读的衬线字体；中文界面保持清晰、克制。
- 排版强调稳定的行宽、行距、段距和页边距，不使用营销式大标题或装饰性卡片。
- 工具栏、选区菜单和侧栏视觉退后于正文，阅读内容始终是第一层级。
- Anx Reader 只作为视觉语言参考，不复制其品牌素材、截图资源或代码。

### 3.2 查词采用 Quill 的悬浮框模式

- 查词框锚定在所选单词附近，并自动避免越出窗口或遮挡关键正文。
- 结果分为：简短中文义、英文定义、当前语境解释。
- 固定动作包括：收藏、复制、重新生成、继续追问。
- “继续追问”不是新开一次性请求，而是把单词、原句、已有释义和阅读位置带入当前书籍的 AI 对话。
- 查词失败、未配置 Provider、超时和取消都必须提供明确状态，不能静默失败。

### 3.3 AI 对话采用可折叠右侧栏

- 右侧栏可展开、收起和拖动调整宽度；收起后不丢失对话。
- 阅读正文随侧栏宽度稳定重排，查词框、工具栏与正文不能重叠。
- 对话按书籍持久化，可保留多个会话；再次打开书籍能够继续原对话。
- 用户可从选词、句子解释或普通输入进入同一会话。
- 每次发送时明确展示正在引用的单词或原文片段，用户可以移除引用。
- 默认只发送完成当前请求所需的上下文，不自动上传整本书。

## 4. 首期范围

### 4.1 必须交付

1. 可复现的 macOS 开发、测试和打包基线。
2. Anx 风格的暖色阅读主题与经过验证的中英文字体组合。
3. 保留并完善 Quill 的语境查词悬浮框。
4. 查词结果一键进入可折叠 AI 侧栏继续追问。
5. OpenAI-compatible、Anthropic、Ollama 的灵活 Provider 配置。
6. 查询历史、词汇收藏和学习状态。
7. EPUB 正文中对已查询或已收藏单词的可选回显标记。
8. 词汇搜索、按书筛选、回到原文、删除和导出。
9. 中英文界面文案、键盘操作、错误状态和基础无障碍。
10. 不含真实凭据和受版权保护正文的测试、文档与发布流程。

### 4.2 首期不做

- 不开发在线书店、账户体系、付费订阅、云端 AI 代理或 DRM。
- 不重写 EPUB/PDF 渲染引擎。
- 不承诺 iPhone/iPad 版本；当前公开 fork 中没有可供修改的原生 iOS 源码。
- 不在首期为 PDF 实现全文自动词汇标记。PDF 保留查词和手动高亮，待 EPUB 方案稳定后单独评估。
- 不加入自动上传整本书的向量库或 RAG。
- 不把《The Alchemist》全文或其他非公版书籍作为示例、测试夹具或发布内容。

## 5. 工程原则

### 5.1 本地优先与隐私

- 书籍、词汇、查询历史和聊天记录默认保存在本地 SQLite。
- API 请求只从 Rust 后端发出，前端不直接持有或记录密钥。
- API Key 和 OAuth Token 只能进入本机 `secrets.db`，不得进入主数据库、同步日志、控制台日志、错误报告或 Git 历史。
- 设置页不得预置任何可用密钥；连接测试和错误信息必须脱敏。
- 用户应能看到并控制每次 AI 请求携带的引用内容。

### 5.2 上游友好

- `origin` 指向 `KlaraGraff/quill`，`upstream` 保持指向 `yicheng47/quill`。
- `main` 尽量保持可追踪上游；个性化功能使用独立、聚焦的功能分支。
- 每个阶段开始前先检查上游变更，避免在已被上游实现的功能上重复开发。
- 不直接修改 `foliate-js`，除非公开 API 无法满足需求；确需修改时使用独立 fork 并记录补丁边界。
- 数据库迁移只能向前兼容，升级前后均不得丢失原有书籍、进度、书签、高亮、聊天和词汇。

### 5.3 产品与代码边界

- 手动高亮与“词汇回显标记”是两个独立概念，数据和渲染不能混用。
- 查询历史与主动收藏分离：查询不等于加入生词本。
- 查词、句子解释和聊天共享 Provider 层，但各自保留适合任务的 Prompt 和输出结构。
- 所有用户可见文案同时维护 `en.json` 和 `zh.json`。
- 非简单功能先写 `docs/features/` 产品规格和 `docs/impls/` 实施计划，再修改代码。

## 6. 目标交互

### 6.1 查词流程

1. 用户选择一个单词或短语。
2. 悬浮框立即显示加载状态，并携带表层词形、所在句、章节和书名请求 AI。
3. 结果依次展示简短中文义、英文定义和语境解释。
4. 查询完成后写入查询历史，但不自动加入收藏。
5. 用户可选择收藏、复制、刷新或继续追问。
6. 点击继续追问时打开现有 AI 侧栏，把本次释义作为可见引用附加到输入区；不得清空既有会话。

### 6.2 书籍理解与追问

- 选中句子或段落后可以进行一次性“解释”，也可以“引用到对话”。
- 侧栏保留当前引用卡片，发送后引用随该条用户消息保存。
- AI 上下文最少包括书名、当前章节和显式引用；扩大上下文必须由用户触发。
- 从历史聊天或释义记录点击“回到原文”时，通过 CFI 恢复到对应位置。

### 6.3 词汇与标记

概念状态至少区分：

| 类型 | 含义 | 默认正文表现 |
|---|---|---|
| 已查询 | 查过但未收藏 | 极轻的点线或下划线，可关闭 |
| 已收藏 / 新词 | 用户主动保存 | 清晰但克制的标记 |
| 学习中 | 正在复习 | 与新词区分的第二种标记 |
| 已掌握 | 用户标记掌握 | 默认不标记，可在设置中显示 |

实现约束：

- EPUB 内容不得被改写；标记使用渲染覆盖层或注释层。
- 只处理当前已渲染章节或可视范围，不能每次翻页扫描整本书。
- 首版匹配至少正确处理大小写和首尾标点。词形还原、短语重叠和跨节点文本作为独立增强项评估。
- 点击词汇标记优先复用历史释义，并提供“重新查询”，避免每次点击都产生费用。
- 提供全局开关，以及按“已查询、已收藏、学习中、已掌握”分别显示的选项。

## 7. AI Provider 方针

### 7.1 支持范围

- OpenAI-compatible：自定义 Provider 名称、Base URL、Model 和 API Key。
- Anthropic：原生协议、Base URL、Model 和 API Key。
- Ollama：本地 Base URL、Model，无强制 API Key。
- OpenAI OAuth：保留为可选实验通道，不作为默认路径；在依赖的非标准后端稳定性得到验证前，不作为发布承诺。

### 7.2 配置目标

- 从单一全局配置逐步演进为“命名 Provider 配置档案”。
- 每个档案记录协议、Base URL、模型和本地密钥引用。
- 允许为查词、解释、聊天分别指定默认模型；首期可先共享一个档案，但数据结构不得阻塞后续拆分。
- 设置页提供连接测试，显示请求目标、模型和脱敏后的错误。
- Provider 不可用时保留用户输入和引用，允许切换档案后重试。
- 测试使用 Mock 服务或本地 Ollama，不在自动化测试中调用付费接口。

## 8. 数据设计方针

编码前需通过独立实施计划确定最终 schema，但概念上至少包含：

- `LookupRecord`：表层词形、标准化词形、结果、语境句、书籍、章节、CFI、Provider/Model、首次及最近查询时间、查询次数。
- `VocabWord`：主动收藏状态、学习状态、复习计数和下次复习时间，并能关联最近一次有效释义。
- `Chat` / `ChatMessage`：沿用按书持久化结构，消息可保存引用文本与 CFI。
- `WordMarkPreference`：全局和按书的标记显示偏好。

不得为同一个阅读位置的每次渲染写入数据库，也不得把自动词汇标记伪装成用户创建的手动高亮。

## 9. 分阶段执行与验收门

### 阶段 0：可复现基线与风险清理

执行内容：

- 核对 remotes、版本和上游同步策略。
- 将 `foliate-js` 子模块地址从 SSH 改为可复现的 HTTPS 地址，并验证固定 commit。
- 安装依赖并记录 Node、pnpm、Rust 和 Tauri 前置版本。
- 运行前端类型检查、lint、build，以及 Rust check、test、clippy。
- 使用公版 EPUB/PDF 建立功能与视觉基线截图。
- 审计 OAuth、日志、`secrets.db` 和示例配置中是否存在凭据泄漏风险。

验收：干净环境可以按文档启动；现有阅读、查词、侧栏、高亮和词汇功能未退化；基线问题有清单，不夹带功能改造。

### 阶段 1：阅读视觉基础

执行内容：

- 形成 Anx 风格的设计 Token、暖色主题和排版规格。
- 选择并核验可再分发的英文字体与中文字体；提供系统字体回退。
- 调整阅读工具栏、正文、查词框和侧栏的视觉层级。
- 覆盖窗口缩放、长标题、长单词和中英文界面。

验收：在常用 Mac 窗口尺寸下正文清晰稳定；控件不遮挡、文字不溢出；提供实施前后截图，由用户确认视觉方向。

### 阶段 2：查词与 Provider 加固

执行内容：

- 固化查词输出结构、加载/错误/取消状态和边缘定位。
- 完成 Provider 档案的最小可用版本和连接测试。
- 保证密钥只保存在本机秘密存储中。
- 为查询历史建立经过迁移测试的数据通路。

验收：OpenAI-compatible、Anthropic、Ollama 至少各完成一次受控验证；切换失败不丢输入；查词结果包含正确语境且可回看；日志中不存在密钥。

### 阶段 3：查词到 AI 侧栏的连续对话

执行内容：

- 完成可折叠、可调整宽度的 AI 侧栏状态管理。
- 增加“继续追问”及可见引用卡片。
- 复用当前书籍既有会话，不因选词或侧栏开关重置历史。
- 保存消息引用和 CFI，支持回到原文。

验收：从查词框进入侧栏不超过一次点击；已有聊天保持不变；关闭和重开侧栏后会话与引用位置可恢复；流式响应和取消正常。

### 阶段 4：词汇闭环与 EPUB 正文回显

执行内容：

- 分离查询历史、收藏和学习状态。
- 在 EPUB 当前渲染内容中增量应用词汇标记。
- 增加显示开关、状态样式、历史释义复用和重新查询。
- 完善按书筛选、搜索、回到原文、删除和导出。

验收：翻页、切章、调整字体和恢复进度后标记仍正确；标记不污染手动高亮；关闭选项后不残留；大型章节滚动和翻页无明显卡顿。

### 阶段 5：复习、可用性与发布准备

执行内容：

- 将现有 `mastery`、`review_count`、`next_review_at` 后端能力接入可用的复习界面。
- 增加批量管理、CSV/JSON 导入导出和数据备份验证。
- 完成键盘、无障碍、空状态、错误恢复和隐私说明。
- 进行回归、视觉、性能、迁移和打包验证。

验收：用户能完成“今日复习 -> 更新状态 -> 回到原文”；导出数据可重新导入；升级现有数据库不丢数据；macOS 安装包可在无开发环境机器上启动。

## 10. 验证矩阵

每个相关阶段至少覆盖：

- 前端：`pnpm exec tsc --noEmit`、`pnpm run lint`、`pnpm build`。
- Rust：`cargo check`、相关单元测试、`cargo test`、`cargo clippy -- -D warnings`。
- 数据：全新数据库、从上游版本升级、失败迁移回滚或明确恢复路径。
- EPUB：分页/滚动、不同字体与字号、章节跳转、CFI 恢复、长章节。
- PDF：查词、手动高亮和侧栏回归；首期不承诺自动词汇标记。
- AI：正常流式响应、超时、取消、401/429/500、无效模型、Provider 切换。
- 视觉：暖色主题、暗色回归、窄窗口、常用桌面尺寸、长中英文文案。
- 隐私：仓库、构建产物、日志、同步事件和导出文件中无 API Key。

## 11. 交付与提交规则

- 每个阶段使用独立分支和聚焦提交，不把多个阶段合成一次大改。
- 提交标题遵循仓库既有约定，例如 `feat(reader): ...`、`feat(ai): ...`、`docs(roadmap): ...`。
- 每阶段交付必须包含：变更摘要、可操作演示、截图、已运行测试、未解决风险和下一阶段建议。
- 未通过当前验收门时不得开始下一阶段。
- 不自动创建 Release，不自动覆盖用户本机已安装的应用。

## 12. 需用户确认的默认决策

本文档提出以下默认值，用户确认总方针即表示同意以它们作为后续详细设计的起点；具体视觉稿仍在阶段 1 单独确认：

1. 功能底座使用 Quill，视觉方向参考 Anx Reader。
2. macOS 桌面版优先，EPUB 自动词汇标记优先于 PDF。
3. 查词写入历史，但只有用户主动操作才进入收藏。
4. 已查询词使用极轻标记；已收藏和学习中使用更明确、可区分的标记；已掌握默认隐藏。
5. AI 侧栏按书保留会话，查词追问复用当前会话。
6. API Key / Ollama 是稳定默认通道，OpenAI OAuth 仅为可选实验通道。
7. 测试仅使用公版书籍和假密钥/本地模型。
8. 每阶段完成后先停下供用户验收。
