## 1. 交付范围

本次实施覆盖：

1. 单词单击打开菜单、双击快速查词，以及短语/句子选区稳定后自动显示菜单。
2. 单词、短语、句子共用的结构化学习卡片。
3. 模块开关、排序、默认展开状态、内容密度、响应式宽度和动态预览。
4. 全书单词自动标记，与手动范围标记严格分离。
5. 纯文本笔记、卡片回显、统一笔记页面、定位原文和完整同步。
6. CEFR 等级、自适应双语和考试成绩估算。
7. 多套完整 AI 服务、服务排序、Key 内部轮换、跨服务故障切换、模型发现和连接测试。
8. EPUB、PDF、TXT/Markdown/HTML 阅读器的触发、定位、错误和无障碍适配。

系统 TTS 按已确认方案延后，不是本次发布门。

## 2. 兼容策略

### 2.1 旧查询记录

- 保留 `lookup_records` 既有字段和读取命令。
- 新增结构化结果 JSON 字段；旧行没有 JSON 时继续使用 definition/context 字段渲染兼容卡片。
- 首次重新查询旧记录时写入结构化结果，不批量调用 AI 回填历史。

### 2.2 旧 AI 配置

- 现有 `ai_profiles` 第一条记录保留为第一套 AI 服务。
- 现有 `ai_credentials` 继续作为该服务内部的 Key 列表。
- 新增服务 CRUD/排序命令，不把 Key 返回 WebView。
- 路由从“单个 active profile”升级为“已启用服务按顺序遍历”。

### 2.3 旧阅读位置

- EPUB/PDF 继续使用 CFI/现有定位。
- 文本阅读器继续使用当前工作区正在升级的源偏移位置格式，并兼容旧 `textloc`。
- 新笔记和标记保存稳定位置标识，不保存依赖当前 DOM 节点的临时引用。

## 3. 数据迁移

新增单向迁移，至少包含：

### 3.1 查询结果扩展

- `lookup_records.result_json TEXT`
- `lookup_records.provider_profile_id TEXT`
- `lookup_records.model TEXT`
- `lookup_records.updated_at INTEGER`

### 3.2 全书单词标记

新增 `word_mark_rules`：

- `id`
- `book_id`
- `normalized_word`
- `display_word`
- `match_mode`，首版固定 `exact`
- `color`
- `enabled`
- `created_at` / `updated_at` / `updated_by_device`

规则不复用 `highlights`，也不为每次出现创建数据库行。

迁移 022 另增 `word_mark_exceptions`，以 `(rule_id, location)` 保存“取消当前标记”的 LWW 状态。`rule_id` 不设数据库外键：同步允许例外事件先于父规则到达，查询通过启用规则联表隐藏临时孤儿；书籍删除仍由显式级联清理。事件与快照校验会验证 `rule_id = hash(book_id, normalized_word, exact)`，防止跨书或跨词挂接。

### 3.3 笔记

新增 `notes`：

- `id`
- `book_id`，全局单词笔记仍记录首次来源书籍
- `anchor_kind`：`word` / `selection`
- `normalized_word`
- `scope`：`book` / `global` / `detached`；删除书籍后选择保留的笔记转为独立资料
- `location`
- `selected_text`
- `content`
- `content_format`，首版为 `plain_text`
- `created_at` / `updated_at` / `updated_by_device`

为 `book_id`、`normalized_word`、`updated_at` 建立查询索引。

### 3.4 学习偏好与考试成绩

- 简单 UI 偏好继续存入 `settings`，模块配置使用版本化 JSON。
- 考试成绩使用独立 `language_assessments` 表，保留考试类型、总分、阅读分数、日期和映射版本。

## 4. 结构化 AI 输出

### 4.1 输出协议

查词和解读请求返回版本化 JSON 语义模型：

- `version`
- `kind`：word / phrase / passage
- `source_text`
- `word_info`
- `context_meaning`
- `target_translation`
- `common_senses`
- `collocations`
- `morphology`
- `grammar_role` / `grammar_analysis`
- `key_terms`
- `idioms`
- `references`
- `reusable_patterns`
- `tone`
- `memory_aid`

字段均允许缺失；前端只渲染启用且有内容的模块。服务不支持严格 JSON 时，后端执行受控解析并返回明确协议错误，不用字符串切段伪造结构。

### 4.2 流式边界

- 后端沿用现有事件流，但累积完整结构后提交最终结果。
- 卡片先显示稳定骨架和加载状态，完成模块可逐步呈现，但宽度不随流变化。
- 结构化请求失败时显示可重试错误，并保留选区和用户输入。
- 旧 definition/context 双流改为一次结构化请求，减少重复内容和费用。

### 4.3 CEFR 提示

- 所有解释请求携带解释模式、CEFR 等级、目标语言、模块配置和密度。
- A1/A2 自适应双语以中文准确义为主，英文部分限制为对应等级。
- B1 英文为主，抽象或易误解部分允许简短中文。
- B2-C2 英文为主，中文是否出现由目标译文模块决定。
- 重点词按语境重要性优先，同等重要时按常见程度排序。

## 5. AI 服务

### 5.1 服务管理

新增命令：

- 列出全部服务。
- 新建、复制、更新、删除服务。
- 启用/停用服务。
- 按折叠卡片拖动结果重排服务。
- 继续复用每个服务内部的 Key 添加、替换、排序和删除命令。

删除服务前先清理其秘密存储引用；删除失败必须回滚，避免数据库与 Keychain 分叉。

### 5.2 路由与健康状态

- 服务按 `enabled, priority, created_at` 顺序候选。
- 每套服务内按 Key 顺序尝试。
- 只有尚未输出内容且错误可重试时才切换 Key 或服务。
- 记录服务和 Key 的错误分类、冷却截止时间、最后成功时间和延迟。
- 请求错误、取消和已经输出后的失败不得盲目轮换。

### 5.3 模型发现

- 使用当前表单草稿 URL、Provider 和选定/首个可用 Key。
- OpenAI-compatible 尝试 `/models` 或规范化后的 `/v1/models`。
- Anthropic/Ollama 使用各自适配器。
- 结果去重、排序后返回；失败仍允许手动输入。
- Key 只在 Rust 后端使用。

### 5.4 连接测试

- 对选定 Model 发起最小真实推理请求。
- 返回成功状态、首次响应耗时、总耗时、服务、模型和脱敏错误。
- 折叠卡片图标触发，不显示文字按钮；提供 tooltip 和 accessible name。

## 6. 阅读交互

### 6.1 单词点击

- 仅正文文本节点中的完整单词触发。
- 拖选、链接、控件、已有手动标记操作和翻页手势不触发。
- 使用成熟的 Unicode 分词能力或浏览器 `Intl.Segmenter`，不以 ASCII 正则作为唯一分词器。
- 单击等待约 240ms 后打开右上方操作菜单，不发送 AI 请求；双击发生时取消待处理单击。
- 双击快速查词可以关闭；关闭后不阻止系统原生双击选词。
- macOS Force Touch 不接管，继续使用系统词典。
- 点击位置生成稳定 location 和上下文句。

### 6.2 操作菜单

- 选区稳定约 150ms 后自动显示；右键和键盘上下文菜单只作为备用入口。
- 单词默认显示“查词”，短语默认显示“释义”，句子/段落默认显示“解读”。
- 菜单根据设置显示问 AI、收藏、标记/取消标记、复制及可选仅翻译。
- 菜单锚定选区右上方并做视口避让。
- EPUB iframe、PDF 渲染文档和直接 DOM 文本阅读器使用统一的 selection DTO。

### 6.3 标记切换

- 选区未被完整覆盖时执行标记，覆盖整个选区。
- 选区完整标记时执行取消；取消子范围时拆分而不是删除范围外内容。
- 已标记单词分别提供“取消当前标记”和“取消全书同词标记”。
- 自动单词规则取消时不触碰手动范围标记。
- 视觉配置进入真实正文渲染，不只作用于设置预览。直接 DOM 文本阅读器完整支持背景、下划线、粗体和字体；EPUB 的手动 CFI 标记与全书同词 CSS Highlight 只应用背景和下划线，不为局部字体或粗体改写出版社正文；PDF 固定版面同样只应用不改变文字层几何的覆盖样式。
- 自定义字体文件仅保存于应用数据目录，通过受限 asset protocol 加载，并在字体列表变化时刷新所有窗口及已挂载的 EPUB 章节 `@font-face`。删除字体同时清理阅读字体与标记配置中的悬空引用。

## 7. 学习卡片和设置

### 7.1 卡片组件

- 建立统一 `LearningCard`，按 word/phrase/passage 渲染模块。
- 模块是全宽分区，不嵌套装饰卡片。
- 必要模块不能关闭，其他模块按设置过滤并排序。
- 底部固定为收藏、问 AI、笔记、复制。
- 已有笔记在操作区上方回显。

### 7.2 尺寸

- 按对齐规格中的类型/密度矩阵计算目标宽度。
- 宽度受阅读区可用空间约束并在一次请求中保持稳定。
- 最大高度 75vh，中间滚动，标题和操作固定。
- 窄窗口转近全宽布局。

### 7.3 阅读辅助设置

- 单词/短语/句子卡片分别配置。
- 模块支持启用、排序、默认展开、密度覆盖。
- 收起后才允许拖动。
- `CircleHelp` 打开只读密度矩阵说明对话框。
- 设置存为版本化 JSON；损坏或未知版本安全回退默认值。
- 设置主体固定宽度；只有“卡片设计”和“操作菜单”在右侧展开独立预览抽屉。

### 7.4 动态预览

- 本地结构化 fixture 驱动即时预览，不调用 API。
- 开关、排序、密度、例句数量、等级和语言即时生效。
- “生成真实预览”显式调用当前 AI 服务，并清楚显示会产生网络请求。

## 8. 笔记和统一查看

### 8.1 命令

- 新建、更新、删除、按书/单词/位置查询笔记。
- 分页列出全部笔记，支持搜索和筛选。
- 返回稳定 location 供“定位原文”。

### 8.2 前端

- 卡片中提供纯文本多行编辑和保存状态。
- 单词笔记提供“本书 / 所有书籍”范围选择。
- 统一笔记页面支持搜索、筛选、编辑、删除和定位原文。
- 收藏与笔记保持独立。

### 8.3 同步

- 新增 note add/update/delete 和 word-mark add/update/delete 事件。
- 合并使用 `(updated_at, updated_by_device)` LWW，删除使用 tombstone。
- 快照完整捕获、应用、确定性导出笔记和标记规则。
- 旧客户端未知事件按现有协议边界处理；迁移和回放测试覆盖重复事件与乱序事件。

## 9. 英语等级与考试估算

- “个人资料与语言”提供手动 CEFR 和成绩估算。
- 首版支持 IELTS、TOEFL iBT、TOEIC L&R、Cambridge English Scale、DET、CET-4/CET-6。
- 映射表版本化并注明官方/近似来源；CET 始终低可信度。
- 阅读分数优先；结果冲突两级以上时返回范围并要求用户确认。
- 用户手动选择始终覆盖估算值。

## 10. 错误、无障碍与隐私

- 图标按钮全部提供 tooltip、accessible name、键盘焦点和加载/成功/失败状态。
- Esc 关闭菜单/卡片；焦点返回触发位置。
- 长单词、长模块标题、窄窗口不得溢出。
- API Key、Provider 原始错误体和书籍大段正文不得进入日志。
- AI 请求只发送当前操作所需上下文。

## 13. 实施结果

### 13.1 迁移与兼容

- 数据库迁移 `021_learning_tools.sql` 与 `022_marker_styles_and_fonts.sql` 已加入迁移链；保留旧查询字段，并以可空 `result_json` 兼容旧历史记录。
- AI 服务继续复用 `ai_profiles` 和系统凭据存储，不把 API Key 写入数据库、同步事件或 WebView 响应。
- 文本阅读器 V2 的源偏移位置、旧 `textloc` 转换和后台准备流程保持兼容；EPUB/PDF 继续使用现有稳定位置格式。
- 同步协议已升级到 v3，同时保留旧版本读取边界；v3 增加单词当前位置例外，笔记、全书单词规则、例外、删除 tombstone 和快照均需覆盖。例外乱序回放使用 `(updated_at, updated_by_device)` 与父规则的同一 LWW 屏障语义，事件、快照和压缩路径必须收敛到相同状态。

### 13.2 已交付行为

- 单击单词菜单、双击快速查词、选区自动菜单、结构化学习卡片、模块顺序/密度/宽度和动态本地预览均已接入。
- 多套完整 AI 服务支持折叠卡片、拖动使用顺序、服务内多 Key、模型发现、最小推理连接测试和双层故障切换。
- 笔记支持搜索、书籍/类型/时间筛选、分页、CSV 导出、编辑、删除和定位原文。
- 删除书籍前可选择删除关联笔记或保留为 `detached` 独立资料；保留操作和删除事件在同一事务中发布，跨设备可重放。
- 手动范围标记与全书单词规则保持独立；手动范围支持完整覆盖切换、部分补齐、同色合并和子范围拆分，全书规则支持当前位置例外。
- CEFR 支持手动覆盖、常见考试估算、成绩历史、冲突确认和 A1/A2 自适应双语提示。

### 13.3 安全与可访问性

- OAuth、服务测试和 AI 路由不向 UI 或日志返回 Provider 原始响应体；错误只保留脱敏分类和必要状态。
- 设置弹窗支持窄窗口上下布局、Esc、Tab 焦点约束和关闭后焦点恢复；图标操作提供 accessible name/tooltip。
- 单词点击使用 Unicode 分词和半开区间命中，避免点击词间边界误触相邻单词。
