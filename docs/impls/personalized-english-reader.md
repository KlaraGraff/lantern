# 实施计划：个性化英语学习阅读器

关联：[总执行方针](../roadmap/personalized-english-reader-master-plan.md)

## 范围

本实施以现有 Quill 阅读器为基础，完成以下连续体验：

1. 安静的暖纸阅读视觉和长篇英文排版。
2. API Key / Ollama 优先的灵活 AI Provider 设置。
3. 查词悬浮框中的“继续追问”，进入同一本书的既有 AI 对话。
4. 自动查询历史与 EPUB 精确位置轻标记。
5. 收藏词的学习状态、到期复习和导出。

## 既有能力复用

- `LookupPopover` 已经并行生成定义和语境解释。
- `AiPanel` 已经可以收起、调宽，并支持携带引用到既有聊天。
- `vocab_words` 已有 `mastery`、`review_count`、`next_review_at`，并被 iCloud 同步事件覆盖。
- Foliate 的注释层可按 CFI 绘制，不需要改写 EPUB 文件。

## 数据边界

### 查询历史

新增本地表 `lookup_records`：保存每次成功触发的查词位置、表层文本、语境、书籍、章节和 CFI。它是阅读轨迹，不等于词汇收藏，也不会与 `vocab_words` 共用行或状态。

查询历史首版保持本地，不进入 iCloud 事件日志。理由是现有同步协议需要同时新增事件、合并、快照和旧端兼容；在未完成整套协议迁移前，不能只把一半数据写入同步层。收藏词及复习状态继续沿用现有可同步通路。

### 正文标记

读取当前书籍的 `lookup_records` 与 `vocab_words`，使用各自存储的 CFI 在 Foliate 注释层绘制：

- 已查询：细点状下划线。
- 收藏新词：柔和金色下划线。
- 学习中：柔和绿色下划线。
- 已掌握：默认不绘制。

标记只附着在读者明确查询或收藏过的精确出现位置；不扫描或改写整本 EPUB，不猜测词形，也不改变手动高亮的数据。遇到与手动高亮完全相同的 CFI 时，手动高亮优先。

## 实施顺序

1. 基线：HTTPS 子模块、锁文件依赖、构建与安全审计。
2. 视觉：暖纸主题、英文衬线字体、侧栏与查词框层级。
3. Provider：默认 API Key、自定义 OpenAI-compatible、前端不再回显密钥。
4. 查词：持久化查询记录、继续追问按钮和 AI 引用卡片。
5. 学习：词汇状态操作、到期复习页、CSV 导出。
6. 验证：迁移测试、前端 build/lint、Rust test/clippy、桌面界面截图。

## 验收

- 查一个词后重开章节，原出现位置仍有可关闭的轻标记。
- 点“继续追问”会打开现有书籍对话并带入单词、原句和当前释义，不清空聊天。
- API Key 不由 `get_all_settings` 返回给 WebView、日志或导出内容。
- 自定义兼容 API 可配置 URL、模型和本地密钥；Ollama 不要求密钥。
- 词汇可标为“学习中”“已掌握”，复习页只显示到期词，操作会更新下一次复习时间。
- 手动高亮、书签、聊天、现有收藏词和迁移前数据库不回归。

## 实施结果（2026-07-12）

- 完成 HTTPS `foliate-js` 子模块基线、暖纸主题、Palatino 默认英文字体及中文系统字体回退。
- 完成安全的 AI 设置读取与保存：WebView 只获取密钥是否已配置，OpenAI-compatible 可使用自定义名称、Base URL、模型和本地密钥。
- 新增本地 `lookup_records` 迁移、查询历史视图和 CFI 回到原文入口；查询历史暂不进入 iCloud 同步，避免不完整事件协议造成跨设备不一致。
- 查词框可将单词、原句、定义和语境解释带入既有 AI 侧栏进行继续追问。
- EPUB 通过 Foliate 注释层显示已查询、收藏、学习中、已掌握标记；手动高亮优先。阅读设置提供四类标记独立开关，已掌握默认关闭。
- 词汇工作区支持到期复习筛选、开始学习、标为掌握、CSV 导出和查询历史浏览。
